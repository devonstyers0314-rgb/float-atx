# Project Retrospective – Float ATX Website

### What Went Well?

>The scope statement was clear and detailed from the start, which made my planning straightforward. There was never any confusion about what the finished project needed to look like, and using GitHub Pages for hosting was the right call free, simple, and it integrates directly with the repo.

>Writing all the website content before touching any code also helped a lot. Knowing exactly what needed to be said made building the HTML structures much easier.

### What Went Wrong?

>Not much, for this project I am the main stakeholder so decisions were fairly easy to make.

### What Would I Do Differently?
>I porbably should commit to GitHub more frequently throughout development, if something breaks late in the project, a clean version history is very valuable.

### Closing Reflection

>The project delivered what it set out to do, a clean professional two-page website that gives Float ATX an online presence before the summer season. The scope was fulfilled, lessons were documented, and the site is live!
